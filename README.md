# Redis Study for Developers

## 📆 스터디 개요

* 일정: 매주 1회 (목요일 기준)
* 진행 기간: 2026년 4월 16일 ~ 2026년 7월 2일 (총 12주)
* 진행 방식: 온라인 (디스코드 예정)
* 최대 1시간 30분 진행

<br>

## 📚 진행 방식

* 매주 **1명 발표**
* 발표자는 해당 주차의 주제를 정리하여 발표 진행
* **나머지 스터디원들도 동일 주차 과제 수행 필수**
* 과제 결과는 **주차별 폴더에 업로드**

<br>

## 📝 스터디 진행 흐름

### 발표

* 발표자가 해당 주차 내용을 정리하여 발표
* Redis 개념 및 동작 방식 설명
* 실무 활용 사례 공유

### 과제 리뷰

* 각자 준비한 과제 내용 공유
* 구현 방식, 트러블슈팅, 학습 내용 토론

### 자유 토론

* Redis 실제 운영 사례
* 성능 이슈 및 장애 사례
* 실무 적용 방법

<br>

## 📂 과제 제출 방식

* 본인 이름 폴더 생성 후 과제 업로드

예시

```
./
 ├ 아무개/
 │  └ [1주차] 마이크로서비스 아키텍처와 레디스.md
 ├ 홍길동/
 │  └ [1주차] 마이크로서비스 아키텍처와 레디스.md
```
